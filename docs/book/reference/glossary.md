# Glossary

> Quick definitions of terms used across the book. One-line definitions with links to the chapter that introduces each term properly.

## What this chapter covers

Alphabetical definitions. Sample entries (the writer fills out the rest):

- **AppStream metainfo** — XML file describing your app to software stores. See [AppStream metainfo](../gnome/appstream.md).
- **Blueprint** — declarative UI language that compiles to GtkBuilder XML. See [Blueprint](../gnome/blueprint.md).
- **Composite template** — a widget class whose contents are defined in `.ui`/`.blp`. See [Declarative UI](../building/declarative-ui.md).
- **DBus** — the desktop message bus. See [DBus with GIO](../system/dbus.md).
- **Flatpak** — sandboxed Linux package format. See [Flatpak and Flathub](../shipping/flatpak.md).
- **GIR** — GObject Introspection metadata. See [Getting started](../foundations/getting-started.md).
- **GObject** — the type system underneath GTK. See [GObject for Python developers](../foundations/gobject-for-python-devs.md).
- **GResource** — bundled-asset format. See [Declarative UI](../building/declarative-ui.md).
- **GSettings** — typed configuration store. See [GSettings](../system/gsettings.md).
- **GVariant** — serialization format used by DBus, GSettings, actions. See [GObject](../foundations/gobject-for-python-devs.md).
- **HIG** — Human Interface Guidelines. See [What makes a GNOME app](../gnome/what-makes-a-gnome-app.md).
- **libadwaita** — the GNOME-platform widget set on top of GTK. See [AdwApplication](../gnome/adw-application.md).
- **Portal** — sandboxed DBus interface for OS features. See [Portals](../system/portals.md).
- **Render node** — a unit in the GTK4 render graph. See [Gsk](../deeper/gsk.md).
- **Symbolic icon** — monochrome SVG that recolors automatically. See [Symbolic icons](../gnome/symbolic-icons.md).
- **Typelib** — compiled GIR; what goi actually loads at runtime.
