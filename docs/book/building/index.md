# Part II — Building GTK apps

The day-to-day mechanics of building a real GTK application. Everything here is pan-platform GTK — it works the same on Windows, macOS, and every Linux desktop. The GNOME-specific layer comes in Part IV.

By the end of this Part you'll have written widgets, laid them out, handled events, built UI declaratively, wired up actions and menus, displayed lists and dialogs, and made async I/O calls without freezing the UI.
